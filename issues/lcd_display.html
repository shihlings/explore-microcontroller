LCD Display Error

Issue

During code execution, the LCD Display will provide the following error randomly:

Traceback (most recent call last):
    File "", line 125, in 
    File "lcd_api.py", line 151, in putstr
    File "lcd_api.py", line 137, in putchar
    File "pico_i2c_lcd.py", line 80, in hal_write_data
OSError: [Errno 5] EIO

*This error message was reproduced by disconnecting the LCD display during runtime.

At the same time, the LCD text would become dim and blurry or even have issues with displaying the correct content.

Solution

At first, I thought the issue was due to unstable connections. I tried swapping in new wires in the second revision of the design. However, it did not work. The issue persisted.

After closely monitoring the LCD Display during runtime, I discovered that the voltage change affects the LCD Display. When the RGB LED Light Strip turns on, the LCD Display becomes dim. I can barely see the text on it. Therefore, I decided to dedicate the whole pin to the LCD Display, which means it won't be sharing the same current with the other devices. It is connected to the Raspberry Pi Pico's V_BUS and GND pins directly. This would guarantee the voltage and current the LCD Display draws are stable and not affected by the other devices.

Dim display. Text not clear.

Dim display. Text not clear and sometimes flashing.

After resolving the issue, the display is bright and clear.
